API Project README

The API project makes use of three APIs, Twitter API, Google Map Javascript API, and Geocoding API. The user searches for tweets based on keyword. And the location of users who have tweeted thoses keyword is plotted on the Google Map. A description of the processes is given as follow.

Twitter API

The user first inputs a keyword search query in the search form which is pased as an input parameter to the Twitter API v2 (Search Tweets - Recent Results). This API fetches the tweets with the matching keywords. It then looks at the user object associated with the tweets, and fetches the "location" property. The Postman Collection was also used as part of this project.

The fetched results is stored in the tweet_result.json file.

GeoCoding API

Based on the "location" property of the user object, the GeoCoding API then tries to retrieve the latlong values for each of the locations. Not all user objects may have this optional property, and those which may have this property would be in a form which will result in an unsuccessful geocode operation.
